Introduction

The product in this case is an assignment completed during the 2015 Web Programming course taught by Professor Ming Chow. I will be evaluting the security of a web server developed by Anna R. As explained by the assignment, Anna used "Heroku, Node.js with the Express web framework, and MongoDB, ... to build a web application that maintains "location check-ins" (login, latitude, and longitude) for a long period of time."

Unfortunately, the server is not secure, and it is my job to find and note the vulnerabilties. It is possible that there are more vulnerabilties then the ones currently listed below, as this is not meant to be a complete professional anaylsis of Anna's server.

Methodology

This code was tested using "black-box" and "white-box" methods. Black-box testing denotes the fact that I attacked the website without the source code using only the URL and publically available tools. I did so, and found those vulnerabilites without use or understanding of the specific source code and its implementation. White-box testing denotes the testing done after I had looked at the source code and was able to preform more informed attacks.

Abstract of Findings

This code was found to be vulnerable to XSS Cross Side Scripting, Overflow of Information, and the falsification of data. This means that it is possible for someone to hijack the user's client using javascript code to run malicious hardware. It also means it is possible to send large amounts of nonsensical data to the sever to overwhelm it, possibly filling the server's memory space and forcing it offline. Lastly, it was noted that it was easy to send the server false data telling it that the client was at locations that did not exist, such as lattitudes in the millions.
